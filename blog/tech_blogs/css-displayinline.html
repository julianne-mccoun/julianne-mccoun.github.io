CSS: Elements Displayed Inline versus as Inline-blocks

This blog post will explain what an HTML element is defined as, a brief explanation of what the display property

The Display Property and Elements

The display property determines if an how an element is displayed - such as whether an element is treated as a block or as an inline element. In HTML, elements are defined by starting tags. Examples of starting tags are the heading tags <h1> - <h6> Some elements have closing tags while others, called void elements, don't need a closing tag (such as image elements).

Examples of Elements

Normal Elements Void Elements
<H1>This is an H1 Heading</H1> <hr> (This element makes a line across your page).
<p>This is a paragraph</p> <img src="your_image_link.png" alt="Alternative text describing your image">

Each element type has a default display value, but these can be overwritten in your code using the property display :. This will only change how it's displayed, not the kind of element. We'll discuss default display types a bit later.

Display: block

A block display starts an element on a new line and takes up the full width that is available. Examples of default display: block include <div>, <h1> - <h6>, and <p> tags.

Look at the images below. Line two of our test.css file sets the display property to block, and on web page image we can see that the navigation bar element takes up the width of the screen, even though the text only takes up about 75% of it. It also takes up the full 80px height determined in line 3 of the code.

Display: inline

An inline display only takes up the space necessary for the content. This means that element does not start on a new line and it will not take up the full height it if is not needed.

Examples of inline elements include: <span>, <a> and <img> elements.

Display: inline-blocks

An inline-block display does not start on a new line. This display creates a block element box (therefore it takes up the full height of 80px even though it is not needed for the content) but it flows with surrounding content as if it were a single inline box.

This allows the developer to set a width and height on the element as well as margins and paddings which is not possible with the display: inline property.
