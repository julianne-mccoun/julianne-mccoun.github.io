Problem Solving

In this blog post on problem solving I'm going to discuss how I worked through the 'Super-Fizz-Buzz' kata challenge for Sprint 4 of foundations and also discuss how I overcomplicated and then simplified the .join() method.

Understanding the Problem: Super-Fizz-Buzz

I think understanding the problem is one of the trickiest parts of coding, and it's an ongoing process. For this challenge I took time to read the question properly and still got it wrong at first. The challenged asked to make a function that takes an argument of an array and returns an array with the following changes:

  • Multiples of 3 as "Fizz"
  • Multiples of 5 as "Buzz"
  • Multiples of 15 with "FizzBuzz"

I solved this task by breaking down the problem into as small of pieces as I could. I developed and created the follwing steps, and I used pseudocode to map out the function at each step:

  1. Make a function that accepts an array argument, declares a new array, pushes each data point from array 1 to array 2, and returns array 2.
  2. All from step 1, but also checks to see if each data point is divisible by 15 and pushes the string "FizzBuzz" instead of the number if so and the original number if not.
  3. All from steps 1 and 2, but also checks if numbers were divisble by 3 or 5 with strings "Fizz" or "Buzz" being pushed if this was the case.

This challenge was very enjoyable because I really got to think through what I was doing and I felt challenged but not totally thrown into the deep end of things and I could work on working the problem rather than be defeated from the beginning.

Some of the helpful tools I used were pseudocoding, the 'rubber ducky method' of talking through the problem out loud and googling things. Test driven development was ideal as I knew what I needed from it before I began writing and could see each time if it works rather than stabbing at it a bunch of times and then seeing what happened.
