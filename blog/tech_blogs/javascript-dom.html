Explain Control Flow and Loops using an example process from everyday life

Control flow is defined as the order in which a computer executes statements in a script. An example of control flow in our everday life is making breakfast: say you want a nice bowl of cheerios, but you haven't had them for a couple of days. Before you get your bowl out, you check to see if you have cheerios. Yes? Great! What about milk? Yes? Nice! Oh wait, is it expired? Nope? Nice! Now you grap out your bowl, pour your cereal and milk, put them both away, grab a spoon and sit down to eat.

You've just completed a process. A mundane one albeit, but good job!

You have a set list of things you do and an order in which you accomplish them in. You even did some variable hoisting, where you checked to see that everything you needed was present and in working order before you set about completing your functions used to make some breakfast.

A programming loop is also something we can relate to breakfast making! Let's talk about pan fried mushrooms this time, because I had those for dinner last night. And let's talk about an if loop, because they're my favorite.

You chop up your mushrooms and put them in a pan with some oil and spices. Here's where the if loop comes in. You look at your mushrooms; they aren't cooked yet, so you miss your if-cooked statement and enter your else statement, which says to stir the pan and wait a minute. This loop repeats for several minutes until, suddenly, they're cooked! you now get to enter the if part of your if statement, which says if cooked, plate and eat! Yay!

The DOM and our Interactions with it

The DOM (Document Obejct Model) is a data representation of objects which make up an element on the web; the most relevant example to me at the moment is a web page. The DOM is primarily used to allow web developers to interact with, change and delete nodes and the content of these nodes. Everything we can change in an HTML element is a node, a few examples being elements and text within elements.

Essentially, the DOM allows Javascript to interact with these nodes that are in HTML and CSS and change them; we can use Javascript this way to bring our website to life. An analogy for this is like marionette puppets. The HTML and CSS make the puppet and how it looks, but Javascript adds the strings to allow it to talk and dance.

HTML elements are selected by tag names, IDs and classes. Depending on the scale we want to reach, we typically use one of these three.

Functions & why we need them

A function in Javascript can be simple or complex, it's all up to the developer! The basic purpose of a function is to group statements togeter to save us time and energy by allowing us to re-use our code! They also help us comply with the DRY principle in programming, which stands for Don't Repeat Yourself.

When creating a function, we declare the word function, the function's name, and then parenthesis after with any arguments for the function. To use our function we "call" it; to do this we simply write the name of our function on a line of code and any arguments for it in the parenthesis next to it. If there are no arguments needed for it, leave them blank. Our function will then run and return any output if it's included in the function.

This use of functions can be applied readily to many applications when you're writing a code. Functions are most useful when the code is repetetive and helps us define a block of code. Inside a function we can do things like declare variables, print console.log() statements, create loops and much more.
